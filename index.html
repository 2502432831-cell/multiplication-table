《九九乘法口诀消消乐》完整 PRD

1. 产品概述

九九乘法口诀消消乐是一款面向小学数学课堂的互动答题活动。教师在活动开始前选择难度、班级与作答学生后，学生通过拖拽乘法算式牌到正确结果牌的方式完成匹配，系统同步统计正确数、得分率、作答用时与历史成绩。

目标用户

小学数学教师、参与课堂互动的学生、需要快速组织课堂竞赛与抽查的教学场景。

核心场景

课堂导入热身、知识巩固、随机抽查、分层练习、课堂竞赛与课后复盘。

1.1 产品定位

本产品定位为“课堂即时互动型数学练习工具”。它不是单纯的题库练习页，而是以课堂即时反馈为核心，强调教师快速发起、学生快速参与、结果快速沉淀。

维度定义
产品类型课堂互动教学活动
知识范围九九乘法口诀与结果匹配
使用方式教师投屏发起，学生上台或触屏作答
价值侧重趣味化练习、课堂控场、过程数据可追踪

1.2 核心价值

  • 提升参与度：用拖拽消除的活动形式替代传统口算问答，降低学生畏难情绪。
  • 提升教学效率：教师只需完成基础设置即可开局，无需额外组织复杂流程。
  • 支持课堂管理：随机点名和指定学生双模式并存，兼顾公平抽查与定向辅导。
  • 沉淀过程数据：通关、结束、历史记录、排行榜四类结果可被教师复盘使用。

2. 功能模块

系统由活动设置、活动作答、结果统计、排行榜与历史、完整 PRD 查看五大模块组成。每个模块都需要同时定义功能描述、功能逻辑与交互逻辑，确保开发实现时无歧义。

2.1 活动设置模块

功能描述

在活动正式开始前，系统以弹窗形式要求教师完成难度、班级、作答模式、作答学生的配置。未完成配置前，活动区不可操作。

功能逻辑

  • 难度共五档：简单 9 题、较易 18 题、普通 27 题、困难 36 题、特难 45 题，默认选中较易。
  • 班级通过下拉选择，当前版本内置默认班级及学生名单。
  • 作答模式分为“随机点名”和“指定学生”，两者互斥。
  • 随机点名模式下必须先完成抽取，指定学生模式下必须完成选择，方可开始活动。
  • 切换班级或切换模式时，需要清空原已选学生，避免错误继承。

交互逻辑

  • 页面首次加载默认展示设置弹窗，并覆盖游戏区域。
  • 点击“随机抽取学生”后，抽中结果展示在当前区域并 Toast 提示。
  • 开始活动校验失败时，Toast 提示明确原因，例如“请先随机抽取学生”。

2.2 活动作答模块

功能描述

活动开始后，中间区域展示算式牌，四周展示结果牌。学生将算式牌拖拽至目标结果牌完成匹配，系统根据匹配关系给出正确或错误反馈。

功能逻辑

  • 系统基于内置题库随机生成本轮算式，按所选难度截取题组。
  • 同一结果可能对应多个算式，但结果牌按结果去重，仅展示一张。
  • 正确拖拽：算式标记为已完成，正确数 +1，算式从界面消失。
  • 错误拖拽：播放错误反馈动画，不移除算式，允许再次尝试。
  • 活动状态分为未开始、进行中、暂停、已完成、已结束。

交互逻辑

  • 拖拽经过结果牌时，结果牌高亮，帮助学生感知投放区域。
  • 暂停后展示蒙层并禁用拖拽；恢复后关闭蒙层并继续计时。
  • 重新开始会重置当前局，但不直接进入活动，而是回到设置弹窗重新配置。

2.3 结果统计模块

功能描述

系统在活动过程中实时记录正确数、得分率和用时，并在通关或中途结束时形成最终记录。

功能逻辑

  • 正确数初始为 0，每答对 1 题累加 1。
  • 得分率 = 正确数 ÷ 总题数 × 100%，结果以整数百分比展示。
  • 计时器在开始时启动，暂停时停止累计，恢复后续计，完成或结束时停止。
  • 全部题目完成后自动弹出通关弹窗，展示学生、班级、难度、得分率、正确数与总用时。

交互逻辑

  • 顶部面板持续显示当前作答学生与过程数据，帮助教师实时观察表现。
  • 通关弹窗内提供“再玩一次”按钮，便于快速发起下一轮。

2.4 排行榜与历史模块

功能描述

排行榜用于展示学生在各难度下的最新成绩；历史模块用于查看某一学生在某一班级和某一难度下的全部历史作答记录。

功能逻辑

  • 排行榜记录维度为：班级 + 学生 + 难度。相同维度保留最新一条作为排行榜展示数据。
  • 历史记录保留同一维度下的全部作答轨迹。
  • 排行榜支持班级筛选、难度筛选和按排名、用时、得分率、正确数、完成时间排序。
  • 中途结束的记录状态标记为“结束”，正常通关记录状态标记为“完成”。

交互逻辑

  • 点击“排行榜”按钮后，以弹窗形式展示记录列表。
  • 点击“查看历史”按钮，打开当前学生的历史记录明细弹窗。
  • 当记录为空时，需要展示明确空状态提示，避免教师误判系统异常。

3. 业务流程

业务流程围绕“发起活动 → 学生作答 → 结果沉淀”展开，既要保证课堂节奏顺畅，也要保证数据闭环完整。

3.1 发起流程

  1. 教师进入活动页。
  2. 系统展示活动设置弹窗。
  3. 教师选择难度与班级。
  4. 教师选择作答模式：随机点名或指定学生。
  5. 若随机点名，则执行抽取；若指定学生，则选择具体学生。
  6. 教师点击“开始活动”。
  7. 系统校验通过后关闭设置弹窗，进入活动进行中状态。

3.2 作答流程

  1. 系统根据难度生成算式牌和结果牌并开始计时。
  2. 学生拖拽算式牌到结果牌。
  3. 系统判断匹配是否正确，并给出成功或失败反馈。
  4. 正确则更新正确数、得分率并移除算式；错误则保留题目继续作答。
  5. 教师可在任意时刻暂停、恢复或结束活动。
  6. 当所有题目匹配完成时，活动自动判定为通关。

3.3 结果沉淀流程

  1. 通关或用户点击“结束活动”时生成一条成绩记录。
  2. 记录需要存储，用于排行榜与学生历史展示。
  3. 排行榜从所有记录中提取“同班级 + 同学生 + 同难度（也就是难度）”的最新成绩进行展示。
  4. 历史记录则按完成时间倒序呈现全部作答轨迹。
  5. 教师可继续发起下一轮活动，实现课堂连续使用。
